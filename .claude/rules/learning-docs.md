# 学习文档落地规则

## 实时更新机制

当对话中出现以下类型的内容时，**主动**判断是否需要落地到 `Learned/` 文档，无需等待用户明确指令：

- 新的知识点、概念解释或技术原理说明
- 问题排查过程及解决方案
- 工具/框架的使用技巧或配置方法
- 架构设计讨论及决策理由
- 用户明确表示"这个要记下来"、"整理成文档"等内容

每次落地更新后，简要告知用户更新了哪个文档的哪个话题。

## 执行流程

### 1. 确定话题名称

从当前对话中提取核心话题，作为文档主题。话题名称应简洁、精准，能够概括对话的核心内容。

### 2. 检查现成文档

检查项目根目录下 `Learned/` 文件夹中是否已存在该话题的文档。

### 3. 新建或追加

- **无现成文档**：在 `Learned/` 文件夹中新建 `[话题名称].md` 文件，按 QA 格式创建初始内容。
- **有现成文档**：在对应话题文档中追加新的 QA 条目，同时更新目录。

## 文档格式要求

所有学习文档采用 **QA 式记录**，必须具备以下结构：

### 文档结构模板

```markdown
# [话题名称]

> 最后更新：YYYY-MM-DD

## 目录

- [Q1: 问题标题](#q1-问题标题)
- [Q2: 问题标题](#q2-问题标题)
- ...

---

## Q1: 问题标题

**日期**：YYYY-MM-DD

**问题**：
[描述问题的背景和具体内容]

**回答**：
[结构化的回答，可包含代码块、列表等]

---

## Q2: 问题标题

**日期**：YYYY-MM-DD

**问题**：
...

**回答**：
...
```

### 格式约束

- **目录必含**：文档顶部必须有目录，每个 QA 条目以锚点链接方式列出，支持按主题点击跳转
- **QA 编号**：按 Q1、Q2、Q3... 顺序编号，新增内容追加到末尾并递增编号
- **日期标注**：每个 QA 条目注明记录日期
- **分隔线**：QA 条目之间用 `---` 分隔
- **代码块**：如有代码或配置片段，使用带语言标注的代码块
- **更新时同步目录**：每次新增 QA 条目时，必须同步更新顶部目录
