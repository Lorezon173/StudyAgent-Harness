## Q1: 用全新 subagent 跑基线测试，为什么测不出"同上下文自评偏向"？

**日期**：2026-06-11

**问题**：
开发 `reviewing-plans-isolated` skill（防止 agent 评审自己刚写的计划时产生偏向）时，按 writing-skills 的 TDD 铁律要先跑 RED 基线——看 agent **不用** skill 时怎么失败。我第一次的设计是：把一份"带作者辩护话术"的计划（"我推荐 A 因为花了很大力气"）喂给一个**全新的 subagent**，让它评审，预期它会被辩护话术带偏。结果它几乎抓全了所有埋雷，RED 没成立。为什么？

**回答**：

**根因：测试装置和被测现象不匹配。** 偏向的物理来源是「**同一个上下文既当作者又当评审者**」——评审者能看到的是**它自己生成方案时的内心辩护链**。而全新 subagent 天然隔离：

- 它读到"我推荐 A 因为花了很大力气"，那只是**别人写在材料里的文字**，不是**它自己的辩护链**。
- 任何 subagent 对它没参与生成的方案都是隔离的，所以用 subagent 跑基线，**物理上就不可能复现同上下文偏向**。

这反向印证了一个关键判断：**真实偏向来自"同上下文共享记忆"，不来自"材料里带了辩护话术"。** 把辩护话术写进材料 ≠ 制造偏向。

**正确的 RED 装置**：让**同一个 agent 在同一上下文里，先生成并辩护一个方案，再回头评审它自己刚写的**。这样评审者才真正"拥有"那条辩护链。改用这个装置后，偏向才复现出来。

**复现出的偏向形状**（比预想更隐蔽，不是"瞎夸"）：同上下文自评会**在作者自己画的框里打磨**——补细节、不撼地基。三个信号：
1. 背书自己的骨架（"骨架可采纳"）；
2. 只复述核心取舍、不独立重算（把"X 是对的"重申一遍）；
3. 接受自己砍掉的范围（"我认可这个克制，不返工"）。

**教训（可迁移）**：设计任何"测试 AI 行为偏差"的实验前，先问一句——**我的测试装置是否真的复现了被测现象的物理来源？** 装置和现象错配时，会得到一个看似干净、实则什么都没测到的"通过"。

## Q2: 把"对照不成立、效应弱"的负向测试信号合理化成"通过"，错在哪？

**日期**：2026-06-11

**问题**：
修正测试装置（见 Q1）后重跑，结果是：复现出的偏向**比预想弱**（Claude 类 agent 本身较自省），且基线方案和重测方案**质量不同、不是严格对照**。我当时用"实质但渐进的提升"这种措辞把它包装成"skill 通过验证"，继续定稿。后来用 skill 自己评审 skill（dogfood）时，隔离评审者一针见血：**"作者把一个负向信号（偏差很弱、对照不成立）合理化成了通过，这是该计划自身偏向最实锤的一处。"** 错在哪？

**回答**：

**双重错误：**

1. **对照不成立 = 混淆实验。** 基线和重测两份方案质量不同，意味着结果差异**无法归因到"用没用 skill"这个变量**上——可能只是方案本身质量不同造成的。一个混淆的实验，无论结果好坏，都不构成证据。

2. **把负向信号合理化成正向结论。** 按 writing-skills 铁律"没有失败的测试就不写 skill"，一个**被混淆且效应微弱**的 RED，根本不是一次干净的失败证据。正确动作是：要么**重做受控对照**（同一份计划，唯一变量是用/不用 skill），要么**诚实标注 "efficacy 未证实" 并降低 skill 的宣称口径**。我却选了第三条——用模糊措辞滑过去。

**最讽刺的一点**：我在开发一个**专门防"在自己画的框里打磨、不质疑框本身"**的 skill，而我合理化弱 RED 的行为，**恰恰就是这个 skill 要治的病**。我对"skill 该不该这样验证"这个框本身没有掀桌子质疑，只在框内找了个说得通的说法。

**教训（可迁移）**：
- **红线信号**：当出现"这样解释就说得通了""实质但渐进""大方向是对的"这类**把负向/不确定信号往正向圆**的措辞时，停下——这往往是在用自洽替代事实（呼应全局规范"探索的终点是穷举不是自洽"）。
- **对照实验的唯一变量纪律**：要证明"X 起了作用"，实验里除了 X 必须其他都一样。两个样本质量不同就比，得到的任何结论都是噪声。
- **自验证的根本局限**：人/agent 验证自己的产物时，最容易在"验证方法本身合不合理"这一层失守，因为质疑它等于否定自己的劳动。**这正是 [[reviewing-plans-isolated]] 隔离评审要补的位**——让一个不拥有沉没成本的外部视角来问"你这验证根本不成立"。
